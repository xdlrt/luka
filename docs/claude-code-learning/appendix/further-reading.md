# 附录 C　延伸阅读

这本书讲的是「设计与原理」，刻意保持了概念上的独立性，不绑定任何具体的技术栈或源码。如果你读完之后想继续深入，下面是几个值得探索的方向。这里不提供具体链接——技术资料更新很快，建议你带着这些关键词，去官方渠道查阅最新的版本。

## 想动手构建一个智能体

如果这本书激起了你「自己也想做一个」的念头，可以从最小的闭环开始：

- 先实现第 1 章的循环——一个能反复问模型、解析工具请求、执行、再问的核心。哪怕只支持「读文件」一个工具，也是完整的闭环。
- 再加上第 4 章的安全关卡——哪怕只有最基础的路径检查和用户确认，也比没有强。
- 然后按第 2、3 章逐步丰富工具和上下文管理。

记住结语的那条克制原则：**从最小可行的闭环开始，把复杂度留到真正需要时再加。**

## 想理解模型的工具调用协议

第 2 章讲的「模型发出工具请求、智能体执行、结果送回」这套机制，背后是各大模型厂商提供的「工具调用」（function calling / tool use）能力。建议查阅你所用模型的官方文档中关于「工具调用」的章节，理解请求和响应的具体格式——尤其是那个贯穿全书的「提问—答复配对」在协议层面长什么样。

## 想了解外部连接的标准

第 6 章提到的几个方向，都有各自的公开规范值得一读：

- **MCP**：一套让智能体连接外部工具服务器的开放协议，关键词「Model Context Protocol」。
- **LSP**：早已广泛使用的代码语义服务标准，关键词「Language Server Protocol」。
- **OAuth**：通用的授权框架，关键词「OAuth 2.0」。

## 想深入安全与权限

第 4 章是全书的核心之一。如果你想把安全做得更扎实，可以了解：

- 操作系统层面的进程隔离与沙箱技术——这是「真正的沙箱」，比书里讲的命令清单严格得多。
- 最小权限原则（principle of least privilege）——一个贯穿整个安全领域的经典思想，和第 8 章「限定子智能体的工具范围」一脉相承。

## 想研究可观测与评估

第 13 章的方向，可以对接到更成熟的工程实践：

- **可观测性**：关键词「observability」「structured logging」「OpenTelemetry」，了解工业级的事件记录与追踪标准。
- **评估**：关键词「LLM evaluation」「eval harness」，了解如何系统地衡量模型与智能体的表现，以及如何避免书中第 13 章警示的那些陷阱。

## 最后

技术的细节会不断变化，但这本书想传递的判断——自主与可控的平衡、代码级的安全、事实高于转述、协议的完整、克制、诚实——是更持久的东西。带着它们去读任何新资料、去看任何新工具，你都会比从前看得更清楚。

愿你既能欣赏智能体的强大，也始终记得为它划好边界。
