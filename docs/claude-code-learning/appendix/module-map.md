# 附录 B　Claude Code 模块地图

这本书是按「设计主题」来组织的，刻意避开了源码细节。但如果你读完之后意犹未尽，想顺着某一章的话题，去翻一翻 Claude Code 这类成熟产品在源码层面大致是怎么分模块的，这份地图可以当一张索引。

它把书中的每一章，对应到该主题在一个编程智能体里通常会涉及的模块方向。这不是任何项目的官方目录，而是一份「想深入时往哪个方向看」的参考。

> **请注意：下表列出的是成熟产品「可能」涉及的可选模块方向，用于指引延伸探索，并不代表本书主体册的论证对象均已具备这些模块。** 凡标注「（成熟产品方向）」的，都属于这一类——它们是产品演进可能走向的形态，而非主体册已论证存在的既有能力。

## 一些偏外围的方向

成熟产品里还有不少更偏产品体验、不直接影响智能体核心闭环的模块，比如：启动引导与初次配置、输出风格主题、网络代理接入、各类运行时工程支撑等。它们大多可以挂靠在上面某个主题之下顺带理解，不必单独深究。读这本书时，把注意力放在前十四章的核心主题上，就足以建立起对一个编程智能体的完整认识。
