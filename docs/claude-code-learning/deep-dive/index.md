# Claude Code 内核解剖

> 编程智能体的硬核实现——《拆解 Claude Code》的深度技术补充本。

这是一本写给资深工程师的深度册子。如果说姊妹篇《拆解 Claude Code》用故事和原理回答「编程智能体大致怎么运转」，那么这本书要回答的是更尖锐的问题：**真实的控制流长什么样？关键的数据结构怎么设计？协议的边界情况怎么处理？安全护栏在代码层面是怎么一层层搭起来的？**

本书可以独立阅读——每章开头都有「前情提要」补齐必要背景；也可以作为科普书对应章节的下钻，每章都标注了它对应科普书的哪一章。

## 阅读前提

- 假设你能读懂带类型签名的伪代码，理解「状态机」「判别联合」「异步生成器」这类概念。
- 不假设你读过任何 Claude Code 源码。机制都会从头讲清楚。
- 每章的硬核程度不同；第 1、4 章最重，建议优先读。

## 关于素材来源与改写方法论（重要）

本书对 Claude Code 内部实现的描述，基于对**一份来源未经核实的源码快照**的研究与改写。请读者明确知悉以下几点：

1. **不逐字复制专有源码**。书中所有用于阐释 Claude Code 机制的代码块，都是为讲解而进行的**结构化重构与改写**（重命名、简化、去除无关分支），用以传达设计思想，而非还原任何具体文件。
2. **类型与流程是阐释性的**。书中出现的类型签名（如 `type StopDecision = ...`）、状态字段、协议消息形态，都是对机制的归纳与再表达，**可能与任何已发布版本存在差异**。凡属推断之处，正文标注「（推断）」。
3. **来源与版本无法核实**。该快照无构建配置、无版本清单，其出处与对应版本均未经核实。本书因此只称其为「一份 Claude Code 源码快照」，不绑定任何具体发布。
4. **可运行的真实代码另有出处**。书中凡标注了来源文件路径（如 `src/harness.ts`）的代码块，均来自一个**独立的、可运行可测试的最小实现**（本仓库的 `luka`），用于展示「同一机制在一个最小系统里如何真正落地」，与上述快照无关。

简言之：**讲 Claude Code 的设计 → 用改写伪代码与类型；讲最小可行落地 → 用带路径标注的真实代码。** 两者在书中始终区分。

> 本书对 Claude Code 内部机制仅作研究性阐释，不构成对任何官方实现、版本或源码真实性的认证。
