# 前言：为什么值得拆解 Claude Code

## 一个会自己干活的程序

几年前，如果你对一台电脑说「帮我把这个登录功能的 bug 修一下」，它只会回你一句「无法识别的命令」。今天，一个叫 Claude Code 的程序可以接住这句话：它会自己打开相关文件、读懂代码、定位问题、改写代码、运行测试，确认修好之后再告诉你「修好了，顺便还补了一个测试」。

这类程序有一个名字：**编程智能体**（coding agent）。它不是聊天机器人——聊天机器人只会说话；它也不是传统的自动化脚本——脚本只会严格执行你写死的步骤。智能体介于两者之间：你给它一个目标，它自己决定每一步做什么，并且能根据中途的结果调整方向。

这件事听起来简单，做起来却牵扯出一连串棘手的问题。模型怎么知道该读哪个文件？它想运行一条危险命令时，谁来拦住它？任务做到一半上下文太长了，怎么办？它改错了代码，怎么发现并纠正？多个任务能不能同时做？出了问题，事后怎么复盘？

Claude Code 是目前把这些问题回答得相当完整的产品之一。拆解它，等于拆解「一个成熟编程智能体应该长什么样」。

## 这本书拆的是「设计」，不是「代码」

市面上讲 AI 工具的内容，要么停留在「怎么用」的教程层面，要么直接跳进源码细节，让没有特定语言背景的读者望而却步。这本书走的是中间一条路：**讲设计，讲权衡，讲为什么**。

我们会反复回到同一个问题：面对某个真实场景，设计者有哪些选择？他们选了哪一个？放弃了什么？代价是什么？这些问题的答案，比任何具体的函数名都更有长久价值——因为工具会过时，但「如何在自动与安全之间权衡」这样的判断不会。

所以你会发现，本书几乎不出现大段代码。我们用流程图、用场景、用类比来解释机制。偶尔出现的技术名词，第一次都会解释清楚，也都收录在书末的术语表里。

## 一条贯穿全书的主线

如果要用一句话概括 Claude Code 的设计哲学，那就是：**让模型尽可能自主，同时把每一个有风险的动作都关进可控的笼子里。**

这条主线会在每一章反复出现，只是换了不同的面孔：

- 在**运行闭环**里，它表现为「什么时候该继续、什么时候必须停下」。
- 在**权限与安全**里，它表现为「代码级的拦截，而不是嘴上叮嘱模型别乱来」。
- 在**扩展能力**里，它表现为「再强的扩展，执行时也得走统一的安全关卡」。
- 在**可观测**里，它表现为「记录足够多的细节用于复盘，但绝不泄露密钥」。

读完全书，你会对这条主线有完整的体感。这也是结语要专门提炼的内容。

## 这本书适合谁

- 对 AI 编程工具好奇，想知道它「到底怎么运转」的技术读者。
- 正在或打算构建类似工具，想先看看成熟设计长什么样的工程师。
- 想理解智能体边界与风险，以便更放心地使用它们的人。

你不需要任何特定语言的编程经验。如果你能看懂「程序会先读文件再改代码」这样的句子，你就能读懂这本书。

下一章，我们先不拆任何单独的部件，而是跟着一次完整的请求走一遍——看看从你按下回车到任务完成，这个智能体内部到底发生了什么。
